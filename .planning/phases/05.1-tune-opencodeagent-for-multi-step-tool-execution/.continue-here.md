<decisions_made>

- qwen3:4b over qwen2.5:3b for grading -- better cross-reference accuracy on checklist evaluation
- Binary checklist decomposition over holistic scoring -- small models hallucinate proportional scores
- Five scoring techniques (A, C, D, E) for dimension-aware weighted aggregation with entailment enforcement
- Section-based classification (rubric headers) over keyword matching (criterion text) -- truly generic
- think:false required for qwen3 -- thinking tokens consume num_predict and break structured output
- num_ctx 4096 for evidence extraction headroom; 300s timeout for CPU inference
- Setup command filtering to prevent base64 blobs from bloating grader transcript
- Technique E (intra-section entailment): if summary criterion TRUE but sibling FALSE, force summary FALSE
</decisions_made>

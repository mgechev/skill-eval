## Problem

The LLM grader (qwen2.5:3b) had good binary discrimination (0.0 for empty, 1.0 for perfect) but poor partial-credit calibration. It hallucinated workflow steps that weren't in the transcript, scored 0.9 when check was skipped, and capped ideal runs at 0.7. Reasoning frequently contradicted transcript content.

## Solution

Replaced holistic 0-1 scoring with binary checklist decomposition informed by three research papers:

1. **CheckEval (EMNLP 2025)**: Binary yes/no per criterion dramatically improves small-model reliability over proportional scoring
2. **RocketEval (ICLR 2025)**: Lightweight LLMs achieve 0.965 human correlation via checklist decomposition with weighted scoring
3. **Autorubric (NAACL 2025)**: CANNOT_ASSESS strategy for prerequisite gating

### Implementation

- **Model**: qwen2.5:3b -> qwen3:4b (better cross-reference accuracy)
- **Prompt**: Enumerate rubric bullet points as numbered criteria; require evidence extraction (commands_found) before evaluation
- **JSON schema**: `{commands_found, criteria[{criterion,met}], reasoning}`
- **Scoring**: Three dimension-aware techniques:
  - **A) Prerequisite gating**: If < 50% workflow criteria met, force efficiency criteria false
  - **C) Weighted scoring**: Workflow criteria count 2x (reflecting 0-0.4 vs 0-0.3 rubric weights)
  - **D) Score cap**: If mandatory workflow criterion is false, cap total at 0.4
- **Config**: think:false, num_ctx 4096, 300s timeout

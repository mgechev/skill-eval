# Plan 05.1-01 Summary: Config Foundation + Screening + Full Validation

## Outcome

**COMPLETE** -- qwen3:1.7b with prompt prefix fix achieves reward 1.00 on both Docker and Local providers, validated with 3 trials each (6/6 perfect). The original Phase 5.1 goal (reward >= 0.85) is exceeded. Plans 02 and 03 are superseded -- no further tuning or Docker-specific validation needed.

Post-fix re-validation (grader memory contention fixes in commits b65a45b, ce9b02c) confirmed perfect scores and ~25-30% faster execution due to exclusive RAM access for the grader model.

## What Was Done

### 1. Config and code changes (Task 1)

- **Model switch**: qwen3.5:4b -> qwen3:1.7b (2x faster, half memory, better /no_think compliance)
- **Tool denial**: 8 tools denied (glob, grep, list, task, todowrite, skill, webfetch, websearch, codesearch, external_directory), 4 allowed (bash, read, edit, write) + doom_loop
- **Prompt prefix**: Added `description` field hint for bash tool calls -- the single biggest fix
- **Env flags**: OPENCODE_DISABLE_CLAUDE_CODE_PROMPT, OPENCODE_DISABLE_PROJECT_CONFIG, OPENCODE_DISABLE_EXTERNAL_SKILLS added to opencode run command
- **Modelfile**: qwen3-1.7b-opencode-agent with num_ctx 8192, num_predict 4096, temperature 0, num_batch 1024, num_thread 8

### 2. Screening experiments (Task 2)

Earlier sessions ran 4 screening experiments (documented in experiments/01-04) covering:
- Qwen 3.5 bug confirmation (tool calls printed as text, not executed)
- Qwen3:4b screening (partial success, then OOM under memory pressure)
- Memory/throughput matrix (qwen3:1.7b identified as sweet spot)
- Prompt eval bottleneck analysis (70% of Ollama time)

### 3. LLM grader memory contention fix

Two fixes resolved intermittent llm_rubric 0.0 scores (grader timeout under memory pressure):
- **Wait-for-unload** (`index.ts`): After agent completes, send `keep_alive: 0` and poll `ollama ps` to confirm the agent model is fully evicted before grader loads
- **num_ctx 2048** (`graders/index.ts`): Lowered default from 8192 to 2048 -- grading transcripts are ~443 tokens, and the larger KV cache caused CPU inference timeouts even with available RAM (attention computation overhead, not memory)

### 4. Docker provider

Docker provider eliminates SIGSEGV entirely (native linux-arm64 on WSL2). Combined with the prompt prefix fix (`description` field hint), first validated trial achieved reward 0.97 in 374s.

### 5. Full validation

Both providers validated with 3-trial runs. Initial validation (pre-grader-fix) scored 0.97-0.98 avg. After grader memory contention fixes (wait-for-unload + num_ctx 2048), re-validation achieved perfect 1.00 across all 6 trials:

## Impact Ranking

1. **Prompt prefix with `description` field hint** -- reward 0.00 -> 0.97. qwen3:1.7b omits the required `description` field in bash tool calls without this hint. One line fixed everything.
2. **Model switch to qwen3:1.7b** -- 2x faster (50 vs 22 tok/s), half memory (1.7 vs 3.2 GB), better /no_think compliance (375 vs 918 tokens). Smaller model, better results.
3. **OPENCODE_DISABLE_* env flags** -- keeps context window clean. Local provider uses fewer commands (6 vs 8) and half the output tokens (220 vs 440) compared to Docker, likely due to cleaner prompt.
4. **Tool denial config** -- reduces cognitive load. Denied tools still get descriptions sent (opencode limitation), but permission denial prevents wasted turns.
5. **Docker provider** -- eliminates SIGSEGV for CI. On local, OPENCODE_BIN_PATH to ARM64 dev binary achieves the same.

## Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| qwen3:1.7b over qwen3:4b | 2x faster, half memory, better compliance, fits 2.6 GiB threshold | Good -- 0.97+ reward on both providers |
| Prompt prefix `description` hint | Model omits required field without explicit instruction | Critical -- sole fix for 0.00 -> 0.97 |
| OPENCODE_DISABLE_* env flags | Prevents loading massive CLAUDE.md/AGENTS.md into 1.7B context | Good -- cleaner execution on local |
| Docker + Local dual validation | Both providers needed for CI and local dev | Good -- both pass consistently |
| Wait-for-unload after agent | keep_alive:0 is async; grader loads while agent model still resident | Critical -- eliminated llm_rubric 0.0 scores |
| Grader num_ctx 2048 default | 8192 KV cache causes CPU timeout even with available RAM; grading only needs ~443 tokens | Critical -- 4x less KV cache, fits within 120s timeout |
| Skip Plans 02/03 | Original problem (model can't complete workflow) is solved; tuning matrix and Docker-specific validation are unnecessary | Efficient -- saved ~2 sessions of work |

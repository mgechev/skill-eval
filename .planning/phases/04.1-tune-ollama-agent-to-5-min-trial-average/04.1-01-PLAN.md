**scripts/benchmark-agent.ts** -- Benchmark runner that wraps the eval CLI.

Structure:
1. Parse CLI args: `--modelfile <path>` (optional, rebuilds model before run), `--env <KEY=VAL>` (repeatable, Ollama env vars -- for documentation only, actual env must be set before Ollama starts), `--trials <N>` (default 3), `--warmup` (default true), `--name <experiment-name>` (required), `--output <path>` (default stdout).
2. If `--modelfile` provided, run `ollama create qwen3-skill-eval-agent -f <path>` to rebuild model.
3. If `--warmup`, run a single trial: `npm run eval -- superlint_demo --agent=ollama --provider=local --trials=1` and discard output.
4. Run measured trials: `npm run eval -- superlint_demo --agent=ollama --provider=local --trials=<N>` with `{ encoding: 'utf-8', stdio: 'pipe' }`.
5. Parse CLI output to extract per-trial data: duration (from "Duration" column, e.g., "440.2s"), reward (from "Reward" column), commands (from "Commands" column). Also extract summary: Avg Duration, Pass Rate.
6. Calculate standard deviation of trial durations.
7. Output JSON result to stdout or `--output` file:
   ```typescript
   interface BenchmarkOutput {
     name: string;
     timestamp: string;
     modelfile: string | null;
     trials: { trial: number; duration_s: number; reward: number; commands: number }[];
     avg_duration_s: number;
     std_dev_s: number;
     avg_reward: number;
     avg_commands: number;
     target_met: boolean; // avg_duration_s <= 300
     reward_met: boolean; // all trials reward >= 0.90
   }
   ```

Use `child_process.execSync` for simplicity (synchronous is fine for benchmarking). Set a generous timeout (900_000ms = 15 min) on execSync. Do NOT modify any OllamaToolAgent code.

Print human-readable summary to stderr so JSON goes clean to stdout. Include: experiment name, per-trial durations, avg +/- stddev, reward, whether 5-min target met.

**scripts/profile-agent-turns.ts** -- Per-turn Ollama metric profiler.

This script does NOT use OllamaToolAgent. It manually runs a simplified agent loop against Ollama's chat API to capture per-turn metrics. It uses the same system prompt, tools, and model as the real agent but with instrumented metric capture.

Structure:
1. Import `Ollama` from `ollama` npm package.
2. Import `AGENT_TOOLS` from `src/agents/ollama/tools.ts` (reuse tool definitions).
3. Load the superlint_demo task instruction from `tasks/superlint_demo/task.toml` (parse the `instruction` field).
4. Run an agent loop (max 15 iterations) sending chat requests with `stream: false, think: false`.
5. After each chat response, capture and log: turn number, eval_count, eval_duration (convert to ms), prompt_eval_count, prompt_eval_duration (convert to ms), load_duration (convert to ms), total_duration (convert to ms), number of tool calls.
6. Execute tool calls using a minimal executor (read_file, write_file, bash via child_process, list_directory). This does NOT need the full permission system -- it's a profiling tool, not production code. Workspace is a temp directory.
7. After the loop completes, print a markdown table to stdout:
   ```
   | Turn | prompt_eval_count | prompt_eval_ms | eval_count | eval_ms | total_ms | tool_calls |
   ```
8. Also print totals and averages.

Important: Do NOT actually execute in superlint_demo workspace (would modify task files). Instead, create a temporary copy of the task workspace using fs-extra.copy, run in the copy, and clean up after. Or use the existing LocalProvider pattern if simpler.

Both scripts should use `#!/usr/bin/env npx ts-node` shebang and be runnable with `npx ts-node scripts/benchmark-agent.ts` and `npx ts-node scripts/profile-agent-turns.ts`.

Do NOT use emojis in console output -- use [INFO], [OK], [WARN], [ERROR] prefixes (same pattern as existing scripts/ollama-bench.sh and tests/benchmark-grader.ts).

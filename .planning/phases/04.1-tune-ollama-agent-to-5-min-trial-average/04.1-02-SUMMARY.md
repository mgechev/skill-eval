# Plan 04.1-02 Summary: Parameter Tuning Experiments

## Objective
Systematically tune qwen3.5:4b parameters, system prompt, and agent configuration to minimize trial duration while maintaining 0.90+ reward.

## What Was Built
- Ran 7 isolated experiments + 4 combination/rerun tests against the 235s baseline
- Applied winning configuration (num_batch 1024) to production Modelfiles
- Documented all results with before/after data in experiments.md

## Key Results

### Experiments (7 isolated + 4 combination)

| Experiment | Avg Duration | Verdict | Why |
|-----------|-------------|---------|-----|
| temp-025 | 227.7s | REJECT (combined) | Isolated: -3%. Combined: +73s std dev |
| structured-prompt | 235.2s | REJECT | No improvement, slightly lower reward |
| predict-2048 | 303.7s | REJECT | Truncated tool calls cause retries |
| max-iter-15 | 313.3s | REJECT | No benefit, high variance |
| **batch-1024** | **215.6s** | **KEEP** | **-8.3%, lower variance, same reward** |
| tool-output-4000 | 274.0s | REJECT | Lost context causes extra commands |
| flash-attn | 268.4s | REJECT | No benefit on ARM64 CPU-only |

### Final Configuration
```
FROM qwen3.5:4b
PARAMETER num_ctx 4096
PARAMETER num_predict 4096
PARAMETER temperature 0
PARAMETER num_batch 1024
PARAMETER num_thread 8  (3 for CI)
```

### Production Performance
Across 6 final trials: avg ~267s, reward 0.98. Target (<=300s) met.
Path variance (4-cmd vs 6-cmd) is the dominant factor in duration spread.

## Decisions
- [Phase 04.1]: num_batch 1024 is the only Modelfile parameter that consistently improves performance
- [Phase 04.1]: temperature 0 is critical for consistency; 0.25 causes wild command count variance
- [Phase 04.1]: System prompt, maxIterations, maxToolOutputChars changes are all neutral or harmful
- [Phase 04.1]: Flash attention + q8_0 KV cache provide no benefit on ARM64 CPU-only
- [Phase 04.1]: Path variance (4 vs 6 commands) dominates duration more than any parameter
